Absolute vs Relative Quaternion:

 

In the case of Unity, the key is to remember that Unity cylinder is defined such that the vertical direction is the y-axis. This means, if we want to rotate a cylinder, we must always construct the rotate operator with respect to the Vector3.up.  OK, here comes the details.

 

Remember, Quaternion, q, is an operator that rotates. So, if I want to rotate the vertical direction of a cylinder from the default direction (Vector3.up) to be aligned with a given newDir:

 

·         I can do:

Vector3 v1 = Vector3.up;  // this is the default vertical-direction of a cylinder

Vector3 Axis = Vector3.Cross(v1, newDir); // newDir is the target direction

float theta = Mathf(Vector3.Dot(v1, newDir)) * MathfRad2Deg;

newRotaiton = q = Quaternion.AngleAxis(thata, Axis);

 

·         Since q is an operator, we are really saying the operator q will rotate from a cylinder’s default vertical direction (Vector3.up) to the new newDir, or, we are constructing the rotation q such that:

dir = q.operateOn(Vector.up);

// Note, “Quaternion.operateOn()” is not defined in the Unity Quaternion class.

// This is to illustrate the effect of q.

And the operator q  will ensure dir is aligned with newDir.

 

·         Now, since the default cylinder vertical direction is defined as Vector3.up axis, I can point myCylinder’s up in the newDir direction by setting q as myCylinder’s rotation:

myCylinder.transform.localRotation = q;

               

At any point the vector myCylinder.transform.up, is the rotation q  from the Vector3.up. So, the following is always true:

myCylinder.transform.up

is  the same as

myCylinder.transform.localRotation.operateOn(Vector3.up)

 

Now, if I want to construct the same rotation with respect to myCylinder‘s current up direction, or compute a rotation from myCylinder.transform.up, to the new newDir, direction,

·         I can do:

Vector3 v1 = myCylinder.transform.up;  // the up-direction of myCylinder

Vector3 Axis = Vector3.Cross(v1, newDir); // newDir is the target direction

float theta = Mathf(Vector3.Dot(v1, newDir)) * MathfRad2Deg;

newRotaiton = q1 = Quaternion.AngleAxis(thata, Axis);

 

·         Equation-A: In this case the q1 operator rotates from myCylinder.transform.up to the new newDir, or, we are constructing the rotation q1 such that:

dir = q1.operateOn(myCylinder.transform.up); 

And the operator q1 will ensure dir is aligned with newDir.

 

·         Equation-B: Remembering that the Unity cylinder is defined with Vector3.up as the vertical axis, then our rotation operator must be defined with respect to the Vector3.up direction, so

myCylinder.transform.up

is  the same as

myCylinder.transform.localRotation.operateOn(Vector3.up)

 

·         The above Equation-A and Equation-B say:

newDir = q1.operateOn(myCylinder.transform.localRotation.operateOn(Vector3.up))

or, this is saying,

·   q1 is defined to operate or myCylinder.transform.up, AND that,

·   myCylinder.transform.up is defined as a transform from Vector3.up

·         So, in order to use q1, I would need to:

myCylinder.transform.localRotation = q1 * myCylinder.transform.localRotation;

We can read this as:

q1 rotate [ myCylinder.transform.localRotation rotate (Vector3.up) ]

                              Again, Vector3.up is significant because it is the default Cylinder’s vertical direction

 

In summary, we have learned that, there are (more than) two ways from rotate a cylinder’s vertical direction to be aligned with a given newDir,

 

·         q rotate from Vector3.up to newDir. We use this by [absolution rotation from initial default value]

o   myCylinder.transform.localRotation = q;

 

·         q1 rotate from myCylinder.transform.up to newDir. We use this by  [relative rotation from current orientation]

o   myCylinder.transform.localRotation = q1 * myCylinder.transform.localRotation;

 

If nothing else, make sure you understand the very last two bullet point when working on the AimLine in MP3.
