10.   Drawing Lines and Planes with Unity

a.       Scene: Cylinder As Line

                                                               i.      Objects:

1.       Cylinder (as the line),

2.       P1/P2: end points of the line (that you can drag in the Editor window when running the game)

                                                             ii.      Script: CylinderAsLine on the Cylinder

         Notice. The default

o    Radius is of end-cap is 1.0

o    Height is 2.0

o    Center position is C = (0, 0, 0)

o    Bottom position P1 is (0, -1, 0)

o    Top position P2 is (0, 1, 0)

         Apply the math we know

o    If Cylinder is a line:

         V = P2 - P1

         Len = V.magnitude

         Vn = (P2 - P1).Normalized

o    Default Vn is (0, 1, 0) (or, Vector.up)

 

p(s) = P1 + s * Vn

 

o    Verify, by default:

p(0) = P1

p(1) = C = (0, 0, 0)

p(2) = P2 + 2 * Vn = P2

 

o    Cool thing, the equation is true for any P1, P2!!

o    In our example, P1, and P2 are under user control, so we compute C to move the Cylinder there, and then scale the cylinder to match the length, lastly, magical for now, we rotate the default cylinder direction, Vector.Up, to align with the line direction V:

V = P2 - P1

C = P1 + 0.5*V; // Center position of the Cylinder

 

// now set the cylinder

transform.localPosition = C ; // Origin is moved here

transform.localScale = new Vector3(r, V.Magnitude * 0.5, r) // r-radius of line

transform.localRotation = Quaternion.FromToRotation(Vector.up, V);

 

b.       Scene: Plane Equation

                                                               i.      Objects: Unity-Plane and Unity-Quad

1.       Plane (or Quad): This is the plane with the equation we will verify

2.       PtFromOrigin: This is the point on the plane from the origin along the Vn (plane normal) direction

3.       PtAlongNormal: This is a position on Vn from the origin further away from PtFromOrigin (use to indicate normal direction)

 

                                                             ii.      Script: DrawPlane on the Plane and Quad

         Notice the default

o    Plane: resting at the origin, normal in the (0, 1, 0) direction (Vector.up)

         Normal = (0, 1, 0), or Vector.up, or transform.up

         D = 0

o    Quad: resting at the origin, normal in the (0, 0, -1) direction (-Vector.forward)

         Normal = (0, 0, -1), or -Vector.forward, or -transform.forward

         D = 0

         Some background knowledge of Unity

o    transform.up/right/forward: are updated continuously reflecting the current y/x/z axis of the object after rotation

         Apply the math we know

o    Since we know the default normal is either Vector.up (Plane) or -Vector.forward (Quad)

o    After user manipulation (rotation): normal vector is

         For Plane: Vn = transform.up

         For Quad: Vn = -transform.forward

o    We know, given P, a point on the plane, it is always true that

         D = P dot Vn

  Interesting fact, transform.localPosition a point on the plane

         D = transform.localPosition dot Vn

o    We know the point on the plane along the normal vector from the origin is:

         PtFromOrigin = D * Vn

o    We can plot a position along the plane normal vector by

         PtAlongNormal = PtFromOrigin + someLen * Vn

         Note: because PtFromOrigin and PtAlongNormal are positions along the Vn direction from the origin, if you move the plane, these two points will not move!
